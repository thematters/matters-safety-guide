# Matters 隱私政策對齊草案

## 文件定位

本文件是安全指南與下一批 NCC 條款調整之間的產品需求，不是已生效條款，也不構成法律意見。現行權利義務仍以 Matters 已發布的服務條款與隱私政策為準。

## 隱私優先原則

### 化名與冒充

- 允許使用化名參與服務，不把公開顯示名稱等同法定身分
- 禁止冒充他人、詐欺與侵害他人權利
- 只有在付款、法定義務、帳號復原或高風險濫用處理確有必要時，才要求額外身分資料
- 介面與條款要明確說明哪些資料會被公開，哪些只供營運或法定目的使用

### 資料最小化與保存

- 每一類個資都要有明確目的、必要性、保存期間與刪除條件
- 不因可能有用而無期限保存裝置、登入或行為資料
- 對安全、防詐與法遵需要保留的紀錄，揭露資料類型、期限與存取權限
- 提供可行的查閱、更正、匯出與刪除途徑，並說明法律或技術上不能立即完成的例外

### 跨境與第三方處理

- 列出主要處理者的角色、資料類型、處理目的與可能地區
- 第三方登入、CDN、email、支付與客服服務分開說明，避免用概括授權包住所有用途
- 重大處理者或目的變更前，提供容易理解的通知與必要選擇

### 付款與公開身分分離

- 清楚說明平台、支付商、銀行與區塊鏈地址各自可能看見的資訊
- 在法律與營運允許範圍內，避免把付款身分直接暴露給其他使用者或不相關內部角色
- 不暗示加密貨幣、公開錢包或化名能自動切斷法定身分關聯

### 公開內容與不可逆保存

- 發布前說明內容會進入的平台資料庫、快取、搜尋、第三方轉載或去中心化保存範圍
- 區分刪除平台顯示、停止散布、搜尋移除與消除所有副本等不同結果
- 對無法保證撤回的內容，在發布前提供具體且易懂的提醒
- 不以「永久保存」或「抗審查」掩蓋對當事人的長期風險

### 帳號安全與事件通知

- 對安全敏感變更採取重新驗證，並提供工作階段檢視與撤銷能力
- 明定疑似帳號入侵、個資事件與重大處理中斷的通知條件、管道與時程
- 讓使用者能找到帳號支援與高風險外部協助，不保證超出團隊能力的事件處理結果

## 發布閘門

1. NCC 條款草案逐項映射本文件，標示採納、延後或不採納及理由
2. 法務、產品、安全與客服共同核對產品行為與文字一致
3. 對付款、跨境、刪除與公開保存做實際資料流盤點
4. 更新 Matters 條款、產品介面、安全指南與客服回覆範本
5. 將可公開重現的結果寫回 `src/data/evidence.json`

在上述閘門完成前，證據儀表板的「化名與個資政策」維持「政策處理中」。
