# 第二輪文字與視覺評估

本輪只處理標題語氣與視覺一致性。研究主張、證據狀態與功能範圍維持不變。

## 評估結果

| 面向 | 原有問題 | 修改方向 |
| --- | --- | --- |
| 標題語氣 | 反問、比喻和口號較多，部分標題像宣傳文案 | 改用名詞短語與直接描述，讓標題說明段落內容 |
| 色彩節奏 | 暖白、全黑和深紫章節接連切換，閱讀時需要反覆適應 | 全站以暖白與淡紫為主要底色，深色集中在少數資訊物件 |
| Hero 構圖 | 左右兩半各有底色，深色圖像又形成第三層舞台 | 移除左右底色切割，讓 Active Trace 成為同一張紙面上的獨立物件 |
| 採訪路線 | 長篇黑底放大緊張感，也讓四段內容顯得沉重 | 改成淡紫閱讀章節，保留深色玻璃圖與捲動切換 |
| 平台來函 | 整段深紫與前一段黑底相連，兩種暗色仍互相競爭 | 改成淡紫底與白色步驟卡，三次來函案例保留深色檔案感 |
| 標題比例 | 多個標題過大，斷行容易落在單一字詞中間 | 收斂二級標題尺寸，重要位置採用人工斷行 |
| 動態 | 動畫本身流暢，問題主要來自暗色背景佔據太長 | 保留 GSAP 與 ScrollTrigger，動畫只使用位移、縮放與透明度 |

## 標題編輯原則

- 標題直接交代主題，不靠反問製造懸念
- 少用「先做 A，再做 B」與「沒有任何工具能」等公式句型
- 任務卡使用動作名稱，避免對讀者喊話
- 證據卡使用措施名稱或目前狀態，不把限制寫成口號
- 人工斷行只放在主要章節，不切斷固定詞組

## 完成後的色彩分工

- 暖白用於 Hero、原則、行動清單、事件處理與研究來源
- 淡紫用於採訪路線、平台來函與平台措施
- 深色只用於 Active Trace、政府來函案例與頁尾
- 螢光綠只保留給狀態、焦點與緊急支援，不再作為大面積背景

## 驗收項目

- 桌機與手機標題沒有孤立單字或不自然斷行
- 採訪路線與平台來函的主標題使用深色文字與淺色底
- Active Trace 仍維持玻璃分層與捲動切換
- `prefers-reduced-motion` 下不產生 pin wrapper
- 主要流程、清單、複製、列印與行動版導覽維持原功能
