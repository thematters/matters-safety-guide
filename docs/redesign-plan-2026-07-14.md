# 2026-07-14 改版設計方案

## 這次要解決的問題

目前頁面有完整的安全知識與證據矩陣，但讀者要自己把內容拼成使用流程。首屏的大標行距太緊，標題字體也沒有表現 Matters 的親近感。部分句子過度工整，讀起來像標語或自動生成的摘要。Active Trace 雖然能動，仍偏向一般風險圖，沒有接住《看不見的平台清道夫》的分層世界觀。

新版把使用者收斂為獨立記者、敏感議題寫作者與消息來源保護者。首頁會沿著一篇稿子的生命週期前進，從查資料、聯絡來源、上稿與收款，一路走到平台收到政府機關來函、文章進入 IPFS 保存，以及讀者改走 Tor 洋蔥入口。

## 核心敘事

首頁主問題改為「一篇稿子，會帶著哪些線索上線？」。這句話同時連到兩個視角。

1. 寫作者視角
   裝置、網路、信箱、互動、檔案中繼資料與金流會留下哪些紀錄。
2. 平台視角
   平台收到調閱要求時，手上實際保存了什麼，能回覆什麼，哪些內容已經進入分散式保存。

## 頁面順序

1. Hero
   用官方 Matters Logo、粉圓體標題與分層 Active Trace 建立主題。
2. 急迫事件捷徑
   保留首屏之後的快速入口，文案改成直接問句。
3. 開始前先說清楚
   把原本三個抽象界線改成化名、服務紀錄與內容撤回三件具體提醒。
4. Active Trace
   依查資料、聯絡與互動、上稿、收款四個工作情境捲動。
5. 平台收到公文之後
   用四步流程與歷史案例說明平台的法律確認、資料盤點、回覆與透明度缺口。
6. 個人行動清單
   保留互動清單，全面改寫工作名稱與說明。
7. Matters 的保護措施與限制
   補入唯讀洋蔥小站、IPFS 發布流程與平台歷史紀錄，所有主張都附證據與限制。
8. 事件處理與研究來源
   保留可操作的緊急步驟，新增韌性創新實驗室兩份手冊。

## 視覺方向

- 標題全面使用 jf open 粉圓 2.1，正文維持系統黑體。
- Hero 與所有多行標題的行高提高，桌機至少 1.02，手機至少 1.08。
- Header 使用官方 Matters 圖形標誌與字標，不再用字母 M 代替。
- Active Trace 改成四片玻璃資料層，分別代表裝置、連線、平台與站外服務。
- 玻璃效果只放在深色的 Active Trace 與平台案例區，保留直角、細線、座標與編輯註記，避免 SaaS 浮卡感。
- 手機版取消背景模糊與固定捲動，只保留自然閱讀順序。
- 平台案例使用《看不見的平台清道夫》既有的 NCC 插畫，建立同一套出版品的視覺承接。

## 動畫方向

- Hero 只有一條入場 timeline，標題直接出現在首屏，不延後 LCP。
- Active Trace 桌機版只有一個 ScrollTrigger timeline，透過標籤切換四個情境。
- 固定的是外層容器，位移與透明度只作用在內部玻璃層。
- 手機不固定，減少動態偏好會關閉 scrub、數字動畫與模糊。
- 動畫只使用 transform、opacity 與 SVG stroke，避免造成版面重排。

## 內容與證據原則

- 洋蔥小站照目前 repo 寫成匿名、唯讀的公開文章入口，不宣稱可登入、發稿或付款。
- 洋蔥小站能降低瀏覽器直接連上公開 Matters 網站與第三方資源的曝露，但 AWS、Matters API、閘道遭入侵與 Tor 流量關聯仍是限制。
- IPFS 依目前 matters-server 的發布流程描述。非垃圾文章發布後會觸發 IPFS 任務，由 Pinata 上傳並把 CID 寫回文章版本。
- IPFS 增加站外取回與抗刪除能力，也提高撤回與更正的難度。付費內容與素材數量過多等情況另有處理，不能寫成每個檔案都永久公開。
- 政府資料請求案例標成 Matters 治理資料中的自述紀錄，不寫成法律意見或對未來個案的保證。
- 證據矩陣繼續區分已驗證、待補證據與政策處理中。

## 外部資料帶來的調整

Resilience Innovation Lab 的《處變不驚》把內容先分成緊急清單、法律環境、常見威脅、裝置、通訊與出入境情境。這個做法支持首頁先提供急迫捷徑，再讓讀者沿工作情境深入。

《Web3 檔案保存入門手冊》同時談到去中心化保存的抗審查價值、閘道仍可能被封鎖、付款與帳號留下的隱私風險，以及永久保存的倫理問題。新版會把 IPFS 放在「保存能力與撤回代價」同一張證據卡中。

## 圖像決策

這一版不新增 GPT 生成圖。現有《看不見的平台清道夫》NCC 插畫已經能準確承接平台、法律與使用者之間的張力，來源與風格也更一致。Active Trace 會改用可讀、可縮放的 HTML 與 CSS 資料層，避免再加一張無法承載互動狀態的裝飾圖。
