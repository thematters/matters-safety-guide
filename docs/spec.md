## 問題與目標

目前頁面已經能說明 Matters 不承諾匿名，也有基本自我檢查，但仍像研究原型。主要問題包括研究面向不完整、勾選分數可能製造安全錯覺、角色切換沒有真正改變建議、平台措施缺乏逐項證據、無障礙與安全標頭不完整、網域與版本責任不清。

本次工作要把網站提升成可正式公開、可持續維護的安全與隱私入口。它應幫助讀者辨識自己的曝險路徑，依情境採取下一步，同時誠實交代 Matters 已做到、尚未做到與仍需政策處理的部分。

## 使用者與任務

### 主要使用者

1. 只閱讀內容、擔心瀏覽紀錄與網路足跡的人
2. 會按讚、追蹤、留言或參與社群的人
3. 發布敏感內容、需要保護身分與素材的人
4. 進行贊助、付款或收款、在意金融資訊連結的人
5. 已遭騷擾、帳號疑似遭入侵或有急迫數位安全事件的人

### 核心任務

- 快速理解「匿名、隱私、安全、抗審查」的差異與限制
- 依閱讀、互動、發布、金流四條足跡找到相關建議
- 建立個人化行動清單，不以總分暗示安全程度
- 查明 Matters 各項措施的證據、狀態、驗證日期與限制
- 在急迫事件發生時，直接找到處置順序與外部協助

## 範圍

### 本次包含

- 單頁式資訊架構重做
- 台灣繁體中文內容重寫與研究補強
- 四種情境的足跡導覽
- 不傳送、預設不保存的個人化行動清單，可由使用者主動複製或列印
- Matters 措施與證據儀表板
- Hero、ScrollTrigger 敘事段落與儀表板動畫
- 鍵盤操作、減少動態偏好與對比度處理
- SEO、分享資訊、sitemap、robots、security.txt
- CSP 與常見安全標頭
- 自動化建置、測試、coverage 與 Codecov
- Cloudflare Pages 發布與線上 smoke test

### 本次不包含

- 修改 Matters 正式服務條款
- 宣稱使用假身分或冒充他人是安全行為
- 帳號登入、後端儲存、分析追蹤或個資收集
- 代替律師、數位安全顧問或緊急救援機構的個案判斷
- 未經平台團隊驗證的技術承諾

服務條款將另隨 NCC 文件批次調整。本站先採較重視隱私的措辭，清楚區分化名與冒充，並避免與現行條款產生直接衝突。

## 體驗原則

1. 情境先於工具。先說明可能的足跡，再建議設定或工具。
2. 證據先於承諾。每項平台措施都標示來源、狀態與驗證日期。
3. 行動先於分數。清單用「現在做、接著做、需要協助」分組，不提供安全分數。
4. 風險語言保持克制。不使用「完全匿名」「保證安全」或「已安全」等絕對語句。
5. 動畫協助理解。所有動態都要對應足跡、狀態或閱讀節奏，且可被減少。
6. 不收集資料。所有互動只在瀏覽器記憶體內運作，重新整理後清空。

## 資訊架構

1. Hero 與核心警語
2. 急迫事件快速入口
3. 四條數位足跡的滾動敘事
4. 個人化行動清單
5. 六個安全主題的知識卡
6. Matters 措施與證據儀表板
7. 外部協助與研究來源
8. 版本、維護責任與資料處理說明

## 功能需求

### 足跡導覽

- 桌面版以 pinned ScrollTrigger 呈現四個章節
- 手機版採自然文件流，不強制 pin 或橫向捲動
- 當章節切換時，示意圖節點、路徑與說明同步更新
- 鍵盤與螢幕閱讀器仍能依文件順序取得完整內容

### 行動清單

- 使用者可選擇一個或多個情境
- 系統依情境顯示帳號、裝置、內容、網路、金流與事件應變任務
- 每項任務分為「現在做」「接著做」「需要協助」
- 勾選狀態僅存在頁面生命週期，不寫入 localStorage、cookie 或後端
- 介面不得顯示總分、百分比或「安全」判定
- 使用者可複製或列印目前清單，網站不得接收結果

### 證據儀表板

- 每項措施需有名稱、狀態、證據來源、驗證日期、限制與負責方
- 狀態只使用「已驗證」「待補證據」「政策處理中」
- 圖表只呈現本站證據集合的真實數量，不模擬平台遙測數據
- 儀表板動畫需以 SVG transform、opacity 或 stroke 呈現

### 急迫事件

- 頁首與頁尾都能進入急迫事件區
- 提供帳號入侵、惡意騷擾、裝置疑似被監控三類處置順序
- 明示何時應停止自行排查並聯絡 Access Now Digital Security Helpline 或可信任的在地支援者

## 內容與研究要求

- 基礎研究來源優先採官方或第一方資料，包括 EFF、CPJ、Access Now、Amnesty International Security Lab、CISA 與 Matters 公開文件
- 每篇建議標示來源或在來源總表可追溯
- 平台能力不能只引用行銷頁，應盡量附 repo 路徑、commit、公開說明或可重現驗證
- 每次發布需更新「最後驗證日期」
- 內容審核需包含隱私、安全、法務語氣與台灣繁體中文校對

## 視覺與動態需求

- 方向採編輯式安全手冊與證據儀表板
- 禁止使用 AI SaaS 常見套版語彙，包括霓虹發光漸層、玻璃擬態卡片、漂浮聊天泡泡、三欄 feature grid、巨大圓角 Bento 牆與無意義 3D 物件
- Hero 使用原生 SVG 足跡地圖，不依賴大型點陣圖
- ScrollTrigger 只用在桌面敘事段落，且需尊重 `prefers-reduced-motion`
- 儀表板動畫不得暗示即時監控或不存在的數據
- 所有動畫以 transform、opacity、stroke 為主，避免 layout thrashing

## 設計系統

- 消費 `thematters/design-system` 的版本化 tokens 與 normal button implementation
- 專案內建立語意別名，元件不得直接使用 primitive brand token
- 自製的敘事圖、狀態卡與資料圖需在設計交付文件記錄為 standalone-site exception
- 最小文字對比須符合 WCAG 2.2 AA

## 隱私與安全需求

- 不載入第三方分析、廣告、追蹤像素或遠端字型
- 不使用 cookie、localStorage、sessionStorage、IndexedDB 或 service worker 保存互動資料
- 外部連結使用安全的 `rel` 屬性
- 設定 CSP、Referrer-Policy、X-Content-Type-Options、Permissions-Policy 與 frame 防護
- 說明 Cloudflare Pages／CDN 仍會為傳輸與防濫用處理一般請求中繼資料
- 不在 repo 或 build 產物放入 secrets

## 無障礙需求

- 全站鍵盤可操作並有可見 focus
- 提供 skip link
- 互動情境控制具正確 button、pressed state 與狀態公告
- 圖表有文字替代，不以顏色作唯一訊號
- 動畫在 reduced motion 下停用 pin、scrub、循環與裝飾性位移
- 主要內容在停用 JavaScript 時仍可閱讀

## 成功標準

- 建置、單元測試、e2e、coverage 與 Codecov 全部通過
- Lighthouse mobile 的 Accessibility、Best Practices、SEO 皆達 95 以上，Performance 達 90 以上
- Hero 不再使用超過 1 MB 的圖片，首次載入不依賴第三方資源
- 所有平台聲明可在 evidence matrix 找到狀態與來源
- 線上網址具正確 canonical、分享資訊、安全標頭與 sitemap
- Pages 網站與 repo commit 可對應

## 主要風險與緩解

| 風險 | 緩解 |
| --- | --- |
| 安全清單造成過度安心 | 移除分數與安全判定，加入情境限制 |
| 平台措施寫得比證據更強 | 狀態分級並公開待補證據項目 |
| 動畫妨礙閱讀或造成暈動 | reduced motion、手機自然流、動畫只增強不承載唯一資訊 |
| 條款後續變更造成文案漂移 | 將條款改版列為獨立政策依賴，本站保持化名與冒充的基本區分 |
| Codecov 或部署整合缺失 | 視為 release blocker，不以本機綠燈代替遠端證據 |
